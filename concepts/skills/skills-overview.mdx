Skills give agents structured, domain-specific behavior without requiring you to wire prompts and tools together every time. Each skill is a directory-based package containing a `SKILL.md` instruction file, optional Python tool scripts, and metadata â€” all discovered and loaded automatically by `SkillLoader`.

## What a skill contains

Every skill is a directory with the following layout:

```text
my-skill/
â”śâ”€â”€ SKILL.md          # Instructions + YAML frontmatter metadata
â”śâ”€â”€ scripts/          # Python files exposing callable tool functions
â”śâ”€â”€ resources/        # Templates, reference assets
```

Only `SKILL.md` is required. The `scripts/` directory is optional but necessary if the skill needs to register executable tools.

## Skills vs. tools

| | Tools | Skills |
|---|---|---|
| **Unit** | Single callable function | Bundle of instructions + multiple tools |
| **Prompt impact** | None | Injects domain instructions into system prompt |
| **Discovery** | Registered manually | Auto-discovered from filesystem |
| **Reuse** | Per-agent wiring | Load by name across any agent |
| **Metadata** | None | `name`, `version`, `author`, `tags`, `requires` |

Skills *wrap* tools. When a skill is loaded, each function discovered in its `scripts/` directory is registered as a tool on the agent, alongside the skill's instructions.

## Loading flow

When an agent loads a skill, the following sequence runs:

```mermaid
graph TD
    INIT[Agent initialized]
    INIT --> RESOLVE[Resolve skill name or object]
    RESOLVE -->|String name| FIND[Search defaults then workspace]
    RESOLVE -->|Skill object| REGISTER
    FIND --> REGISTER[Register skill on agent]
    REGISTER --> TOOLS[Append tool schemas to internal_tools]
    TOOLS --> EXEC[Store executors in skill_tool_executors]
    EXEC --> PROMPT[Rebuild system prompt with skill instructions]
    PROMPT --> READY[Skill active in chat loop]
```

## Quick start

```python
from logicore.agents.agent import Agent

agent = Agent(
    llm="ollama",
    tools=True,
    skills=["web_research"]
)

response = await agent.chat("Find and summarize recent updates on AI safety")
print(response)
```

<Tip>
  Use small, focused skills per domain capability. A skill named `web_research` that does one thing well is easier to maintain and combine than a broad skill trying to cover many domains.
</Tip>

## Why skills matter

- **Consistency** â€” the same instructions and tool bundle are reused across projects and team members, eliminating prompt drift.
- **Faster setup** â€” load a named skill instead of wiring tools and prompts individually every time.
- **Better quality** â€” domain-specific guidance in `SKILL.md` helps the model follow the correct workflow for a given task.
- **Governance** â€” `SkillMetadata` fields (`name`, `version`, `author`, `tags`, `requires`) make capabilities auditable and versioned.
- **Composability** â€” multiple focused skills can be combined on a single agent for well-rounded capability.
